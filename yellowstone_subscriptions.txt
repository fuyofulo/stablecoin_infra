1. accounts

  - stream account updates
  - filters available:
      - account: exact account pubkeys
      - owner: owner program pubkeys
      - account-state filters:
          - memcmp
          - datasize
          - token_account_state
          - lamports comparisons
      - nonempty_txn_signature

  2. transactions

  - stream full transaction updates
  - filters available:
      - vote
      - failed
      - signature
      - account_include
      - account_exclude
      - account_required

  3. transactions_status

  - stream transaction status updates without the full transaction payload
  - uses the same filter shape as transactions:
      - vote
      - failed
      - signature
      - account_include
      - account_exclude
      - account_required

  4. slots

  - stream slot updates
  - slot-level stream, useful for chain progress / lag tracking

  5. blocks

  - stream full block updates
  - filters available:
      - account_include
      - include_transactions
      - include_accounts
      - include_entries

  6. blocks_meta

  - stream block metadata updates
  - lighter than full blocks
  - useful if you want block timing / slot progress without full block payloads

  7. entry

  - stream ledger entry updates
  - very low-level stream

  8. ping

  - keepalive / heartbeat support

  And these request-level controls also exist:

  9. commitment

  - choose commitment level, e.g. processed, confirmed, finalized

  10. from_slot

  - replay / backfill starting from a given slot

  11. accounts_data_slice

  - request only byte slices of account data instead of full account payloads
